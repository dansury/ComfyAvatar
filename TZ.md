## Задача
Создай полностью рабочий сервис для локальной генерации AI-аватаров с веб-интерфейсом. Сервис должен автоматически находить существующий ComfyUI (включая portable версию) на компьютере, использовать его как бэкенд и предоставлять простой UI для анимации фото с озвучкой.

## Технические требования

### Архитектура
1. **Backend**: Python + FastAPI
2. **Frontend**: React (или простой HTML/JS с Tailwind, если проще)
3. **ComfyUI Integration**: Автоматическое обнаружение + REST API общение
4. **Хранение данных**: localStorage (куки) для настроек + история генераций в JSON файле

### Автоматическое обнаружение ComfyUI
Сканируй следующие пути для поиска ComfyUI:
- Windows: `C:ComfyUI`, `C:ComfyUI_windows-portable`, `%USERPROFILE%ComfyUI`, `%APPDATA%ComfyUI`
- Portable версии: ищи `python_embeded/python.exe` или `run_nvidia.bat`
- Если найден portable — используй его встроенный Python
- Запускай ComfyUI автоматически если не запущен (порт 8188)

### Основной функционал
1. **Загрузка фото**: drag & drop или кнопка, валидация (формат, размер)
2. **Запись/загрузка голоса**: 
   - Запись через микрофон (5-15 секунд)
   - Или загрузка WAV/MP3 файла
3. **Ввод текста**: текстовое поле для инструкции
4. **Генерация TTS**: Использование Coqui XTTS v2 или Kokoro TTS для клонирования голоса по образцу
5. **Генерация видео**: Запуск SadTalker workflow в ComfyUI с аудио на вход
6. **Прогресс**: WebSocket для отображения прогресса генерации
7. **История**: Сохранение всех генераций (фото, аудио, видео, timestamp) в `history.json`
8. **LocalStorage**: Сохранение настроек (путь к ComfyUI, последние параметры)

### UI требования
- Простой, интуитивный интерфейс
- Секции: "Загрузка фото" → "Голос" → "Текст" → "Генерация" → "Результат"
- Отображение прогресса (spinner + текст)
- Предпросмотр результата с кнопками "Скачать", "Повторить"
- История генераций снизу (карточки с превью)
- Адаптивный дизайн (работает на десктопе и планшете)

### Workflow для ComfyUI
Используй **SadTalker** workflow:
1. Загрузи фото в LoadImage node
2. Загрузи аудио в LoadAudio node  
3. Подключи к SadTalker节点
4. Сохрани результат в SaveVideo node
5. Возвращай URL видео через API

### Функции backend
1. `find_comfyui()` — автопоиск ComfyUI (порты, пути, portable detection)
2. `start_comfyui_if_needed()` — запускает если не запущен
3. `generate_tts(text, voice_sample)` — генерирует аудио через XTTS v2
4. `run_sadtalker_workflow(photo, audio)` — отправляет в ComfyUI, ждёт результат
5. `get_history()` — возвращает историю генераций
6. `save_settings()` / `load_settings()` — localStorage + файл

### Особенности
- **Portable ComfyUI**: если найден `python_embeded/`, используй его вместо системного Python
- **Ошибка подключения**: если ComfyUI не отвечает, пробуй перезапустить с портом 8188
- **Обработка ошибок**: красивые сообщения на UI при ошибках
- **Кеширование**: кешируй найденный путь к ComfyUI в settings.json

### Дополнительные требования
- Локальный запуск без интернета (все модели локально)
- Поддержка русского языка в UI
- README с инструкцией по запуску: `python -m backend.main`
- Зависимости: `fastapi`, `uvicorn`, `requests`, `websockets`, `torch`, `coqui-tts`
- Все баги выводятся в лог и в веб-интерфейс, чтобы легко было отправить тебе и пофиксить
- Start.bat или другой файл не должен закрываться при ошибках, а должен ждать пока я все не увижу и не отправлю тебе, если что-то не запустилось или не скачалось
р Загрузка компонентов должна сохранять прогресс, то есть если отвалился интернет, надо чтобы загрузка продолжалась, а не начиналась заново. если для этого надо сделать установку через браузер или как-то еще, так и сделай.

1. Запускается одной командой
2. Автоматически находит ComfyUI
3. Предоставляет простой UI для генерации аватаров
